# Deployment

This project has two deployable pieces:

1. **The comment server** — a Node.js/Express app that stores and serves comment data
2. **The static docs** — plain HTML files generated by `build.js`, hosted anywhere

## Comment server

The comment server needs to be always-on and accessible from the browser. It serves the comment API and the static assets (`app.js`, `sidecar.css`) that power the comment UI on every page.

For this demo, the comment server runs at `https://comments.detz.dev`.

The server is deployed using Docker behind a Caddy reverse proxy. See the [detz.net-caddy](https://github.com/ryanrdetzel/detz.net-caddy) repo for the infrastructure config.

### Environment

The server needs no special environment variables. It creates `comments.db` in the working directory on first run.

```bash
# Install and start
npm install
node server.js
```

For production, run it with a process manager or as a Docker container:

```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --omit=dev
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]
```

### CORS

By default the server allows all origins. For production, restrict this to your docs domain in `server.js`.

## Static docs (GitHub Pages)

The `docs/` directory contains the markdown source files. A GitHub Actions workflow builds them into HTML and deploys to GitHub Pages automatically on every push to `main`.

### Required secret

Before the workflow can run, add a `SITE_ID` secret to your GitHub repository:

1. Generate a stable ID: `node -e "console.log(require('crypto').randomUUID())"`
2. Go to **Settings → Secrets and variables → Actions → New repository secret**
3. Name: `SITE_ID`, Value: the UUID you generated

Keep this value stable. Changing it reassigns all document IDs and orphans existing comments.

### GitHub Pages setup

1. Go to **Settings → Pages**
2. Set **Source** to `GitHub Actions`

The workflow handles the rest.

### Build command

The CI runs:

```bash
node build.js \
  --input ./docs \
  --output ./dist \
  --server https://comments.detz.dev \
  --site-id $SITE_ID \
  --assets-url https://comments.detz.dev
```

The `--assets-url` points to the comment server because that's where `app.js` and `sidecar.css` are served from. This means every generated page loads the latest frontend code from the comment server — no need to redeploy the static site when the UI changes.

## Self-hosting on a different domain

To use your own comment server:

1. Deploy the server somewhere with HTTPS
2. Update the build command to point `--server` and `--assets-url` at your server
3. Update the `SITE_ID` secret (or keep it if you're migrating an existing deployment)

The static docs can live on any static host — GitHub Pages, Netlify, Cloudflare Pages, S3 + CloudFront, etc. They make cross-origin requests to the comment server, so ensure CORS is configured correctly on the server.
